# Product Adoption Guide

## 목적
이 저장소는 단순한 문서 사이트가 아니라, 앞으로 만드는 여러 제품이 공통으로 참고할 수 있는 웹 기반 디자인 스타일 가이드이자 레퍼런스 구현체다.

즉, 여기서 보여주는 토큰, 폰트, 아이콘, 컴포넌트, 레이아웃 패턴은 실제 제품 코드에 그대로 옮겨도 되는 기준이어야 한다.

## 운영 원칙

### 1. 토큰이 소스 오브 트루스다
- 색상, 타이포그래피, 간격, radius, shadow, transition은 토큰으로만 관리한다.
- 컴포넌트는 토큰을 소비하는 형태로 만든다.
- 제품은 컴포넌트를 복사하더라도 토큰 체계만 유지하면 같은 결과가 나와야 한다.

### 2. 문서 예제는 실제 제품 코드처럼 보여야 한다
- 데모 전용 편법보다 제품에서 그대로 쓸 수 있는 구조를 우선한다.
- 예제에서 쓰는 마크업, 클래스, 상태 패턴은 실제 서비스 구현 방식과 최대한 같아야 한다.
- 문서용 장식과 제품용 기준이 다르면 안 된다.

### 3. 폰트는 토큰으로 제어하고, 외부 의존은 최소화한다
- 한글/영문 폰트 선택은 컴포넌트 내부가 아니라 전역 토큰으로 제어한다.
- 현재 기준:
  - 한국어 모드: `Noto Sans KR`
  - 영어 모드: `Roboto`
  - 숫자/코드: `Roboto Mono`
- 제품 관점에서는 폰트 사용처보다 폰트 토큰을 먼저 맞추는 것이 중요하다.

### 4. 아이콘은 제품 기준으로는 로컬 SVG가 기본이다
- 제품에 들어가는 아이콘은 외부 아이콘 폰트 CDN 의존보다 로컬 SVG 방식이 유리하다.
- 이유:
  - 네트워크 실패 시 레이아웃 붕괴를 줄일 수 있다.
  - 아이콘 품질과 크기 제어가 쉽다.
  - 접근성과 번들 관리가 더 명확하다.
- 문서 사이트가 당장 외부 폰트를 일부 쓰더라도, 최종 제품 기준은 SVG 우선으로 가져간다.

### 5. 다국어와 테마는 옵션이 아니라 기본 조건이다
- KO/EN 전환은 페이지 단위가 아니라 시스템 전역 상태로 관리한다.
- Light/Dark도 토큰 기반으로 동작해야 한다.
- 새 컴포넌트와 새 패턴은 처음부터 언어/테마 전환을 깨지 않게 작성한다.

### 6. 패턴까지 포함한 디자인 시스템이어야 한다
- 이 저장소는 버튼, 인풋 같은 개별 컴포넌트만 다루지 않는다.
- 실제 제품 화면 패턴까지 포함한다.
- 기본 패턴 순서:
  1. Filter / Control
  2. Summary / KPI
  3. Main View
  4. Detail / Table

## 기술 기준

### 폰트 전략
- 문서 사이트 단계에서는 Google Fonts CDN 사용 가능
- 단, 폰트 선택은 반드시 전역 토큰으로 연결
- 향후 제품 배포 안정성이 더 중요해지면 self-host 전환 가능
- 제품이 여러 개로 늘어날수록 `폰트 로딩 방식`보다 `토큰 일관성`이 더 중요하다

### 아이콘 전략
- 목표 기준: 로컬 SVG
- 권장 방식:
  - 공통 `Icon` 컴포넌트 또는
  - 공통 SVG sprite / assets 폴더
- 지양:
  - ligature 기반 아이콘 폰트에 전면 의존
  - 아이콘 CDN 실패 시 텍스트가 노출되는 구조

### 컴포넌트 전략
- 스타일 가이드는 `문서`이면서 동시에 `레퍼런스 구현`이어야 한다.
- 컴포넌트 API와 상태 이름은 실제 제품에서 재사용 가능한 수준으로 유지한다.
- 데모용 임시 네이밍보다 제품에서 오래 버틸 이름을 선택한다.

## 앞으로 이 저장소에서 우선할 방향
1. 토큰 체계를 계속 정리한다.
2. KO/EN, Light/Dark를 모든 주요 페이지에 일관되게 적용한다.
3. 아이콘 시스템을 로컬 SVG 기준으로 전환한다.
4. 문서 예제를 실제 제품 복사 가능한 수준으로 유지한다.
5. 필요하면 이후에 폰트도 self-host 가능한 구조로 옮긴다.

## 결론
이 프로젝트는 `보기 좋은 데모 사이트`보다 `앞으로 여러 제품이 참고할 기준 시스템`에 더 가깝게 운영한다.

따라서 현재 기준 추천은 아래와 같다.
- 폰트: 토큰 기반, 현재는 CDN 사용 가능
- 아이콘: 장기적으로 로컬 SVG 전환
- 컴포넌트: 데모가 아니라 제품 레퍼런스 기준으로 유지
- 패턴: 실제 운영 화면 구조까지 포함
